## 🎯 資料夾目的

這是**非同步跨團隊溝通的中樞**。

由於各團隊可能在不同的終端 (Terminal Session) 執行，無法即時對話。
此資料夾作為**檔案型訊息佇列 (File-based Message Queue)**，
讓團隊間能透過放置/掃描文件來完成跨團隊協作請求。

**核心原則**:
- 📄 每個請求 = 一個 `.md` 檔案
- 🔍 團隊啟動時掃描此資料夾，處理與自己相關的請求
- ✅ 處理完成後在檔案內標記完成
- 📦 完成的請求定期歸檔

---

## 📝 檔案命名規範

```
{YYYYMMDD}_{HHMM}_{FROM}_to_{TO}_{short_desc}.md
```

### 命名範例
```
20260306_1430_T01_to_T02_kv_field_alignment.md
20260306_1500_T03_to_T06_error_msg_review.md
20260306_1600_M01_to_ALL_stage1_qa_meeting.md
20260307_0900_T02_to_T01_new_field_request.md
```

### TO 欄位選項
| 值 | 含義 |
|----|------|
| `T01` ~ `T07` | 指定單一團隊 |
| `M01` | 專案領導 |
| `ALL` | 所有團隊 (廣播) |
| `T01_T03` | 多團隊 (用底線連接) |

## 🔄 狀態生命週期

```
🟡 PENDING          來源團隊建立請求
    ↓
🔵 ACKNOWLEDGED     目標團隊已看到,開始處理
    ↓
✅ COMPLETED        目標團隊處理完成,來源團隊確認
    ↓
📦 ARCHIVED         M01 定期歸檔到 archive/ 子資料夾

特殊狀態:
❌ REJECTED         目標團隊無法處理 (需說明原因)
⏸️ BLOCKED          等待其他前置作業
```

## 🔍 團隊掃描 SOP (每次啟動 Session 必做!)

### Step 1: 掃描與自己相關的請求
```bash
# (概念範例)
# 如果你是 T02,掃描所有目標包含 T02 的檔案
ls shared/cross_team_comms/*_to_T02_*.md
ls shared/cross_team_comms/*_to_ALL_*.md

# Glob 模式
# *_to_{MY_TEAM}_*.md
# *_to_ALL_*.md
```

### Step 2: 檢查檔案內部狀態
- 打開每個符合的檔案
- 檢查 `**狀態**` 欄位
- 如果是 `🟡 PENDING` → 進行處理

### Step 3: 處理請求
1. 將狀態改為 `🔵 ACKNOWLEDGED`
2. 執行請求內容
3. 在「回應區」填寫處理結果
4. 將狀態改為 `✅ COMPLETED`
5. 如果需要進一步協調 → 建立新的請求檔案

### Step 4: 歸檔 (M01 負責)
- M01 定期將 `✅ COMPLETED` 的檔案移動到 `archive/` 子資料夾

## ⚠️ 使用規則

### ✅ 必須遵守
1. **啟動時必掃描** — 每次開啟新 Session,第一件事就是掃描此資料夾
2. **填寫完整** — 建立請求時必須填寫所有必要欄位
3. **及時回應** — 看到 PENDING 請求後應盡快 ACKNOWLEDGE
4. **狀態同步** — 處理完畢後必須更新狀態
5. **不刪除** — 即使請求已完成也不刪除,由 M01 統一歸檔

### ❌ 禁止事項
1. ❌ 不得刪除他人建立的請求檔案
2. ❌ 不得修改請求內容區 (只能修改回應區和狀態)
3. ❌ 不得在此資料夾放置非請求類型的文件
4. ❌ 不得繞過此機制直接修改其他團隊的文件
