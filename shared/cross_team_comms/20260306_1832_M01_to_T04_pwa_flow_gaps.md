## 請求內容

UX 使用者分析師發現 PWA_SPEC.md 存在 **9 個使用者流程缺口**。請 T04 更新文件補齊。

### 🔴 HIGH — 流程斷點

**H1: 缺少 Login 頁面**
- PWA_SPEC.md 定義 6 個頁面，無 Login 頁
- 使用者找不到登入入口，無法參與分析和獎勵
- 需定義: 匿名可瀏覽 → 點擊「分析」時觸發 OAuth → 登入後返回

**H2: 冷卻期 UI 未定義**
- T05 定義連續 3 次失敗觸發 1 小時冷卻
- 前端沒有冷卻倒數計時器、按鈕禁用狀態、i18n 字串
- 使用者只看到通用「請求過於頻繁」，不知要等多久
- 需新增: `reward.cooldown.active` / `reward.cooldown.remaining` i18n key

**H3: 72 小時分析時限無預警**
- 文章超過 72 小時不接受分析（T03 Layer 4）
- 文章卡片和詳情頁沒有「可分析」/「已截止」狀態標記
- 使用者可能分析完才被拒絕，浪費時間
- 需新增: 剩餘可分析時間顯示、超時後禁用分析按鈕

**H4: IndexedDB `model_files` store 未建立**
- PWA_SPEC.md L300-355 只建立 4 個 store
- L740-741 卻使用 `model_files` store（OPFS fallback）
- 會導致模型儲存失敗，使用者重複下載 2GB
- 需在 `onupgradeneeded` 新增 `model_files` store

### 🟡 MEDIUM — 體驗不完整

**M1: 無 Onboarding 引導**
- 新使用者不知道 App 做什麼、光譜條代表什麼、如何參與
- 建議: 3-5 步首次使用引導（概念→光譜教學→示範→登入）

**M2: 低階裝置推理等待體驗不足**
- 只寫「思考中...」，無百分比、預估時間、超時 fallback
- 建議: 動畫進度、>10s 安撫訊息、>30s 提供伺服器模式按鈕

**M3: 離線提交缺少即時回饋**
- 使用者離線提交後不知分析是否已保存
- 建議: toast「已保存，連線後自動提交」+ Profile 顯示「待同步」

**M4: Background Sync 失敗後靜默丟棄**
- MAX_SYNC_RETRIES=5 超過後「通知使用者」但通知在哪裡？
- 建議: App 開啟時檢查被丟棄項目，persistent toast + 重新提交選項

**M5: 隱私政策同意 UI 缺失**
- PRIVACY_POLICY.md 要求首次登入時取得同意（個資法第 8 條）
- PWA_SPEC.md 沒有同意彈窗、checkbox、同意按鈕
- 建議: OAuth 前插入隱私政策同意步驟，同意紀錄寫入 KV
