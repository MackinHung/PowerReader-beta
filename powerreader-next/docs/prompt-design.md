# Prompt Design — 媒體識讀分析參數

> PowerReader 雙 Pass 分析架構的參數設計文檔

## 設計原則

1. **少則精** — 每個參數必須對媒體識讀有直接幫助
2. **JSON 格式鎖定** — prompt 尾部附完整 schema，約束 LLM 輸出
3. **光譜兩端定義** — 只定義 0 和 100 的極值，中間由 LLM 自由判斷
4. **代碼能解決的不進 prompt** — 例如 `source_attribution` 由代碼從 `article.source` 組裝

## 參數清單

### Pass 1：量化分析（Score Extraction）

| 參數 | 型別 | 回答的問題 | 媒體識讀價值 |
|------|------|-----------|-------------|
| `is_political` | bool | 這篇跟政治有關嗎？ | 輔助：決定 UI 顯示模式（非政治 → bias 固定 50、隱藏光譜） |
| `bias_score` | 0-100 | 這篇偏綠還是偏藍？ | **核心**：政治光譜定位，0=極綠 ←→ 100=極藍，50=中立 |
| `camp_ratio` | {green, white, blue, gray} | 誰的聲音在主導？ | **高**：看出單一觀點壟斷或多元呈現，四值加總=100 |
| `emotion_intensity` | 0-100 | 文章在煽動我嗎？ | **高**：辨識情緒操控手法，0=冷靜客觀 ←→ 100=極端煽動 |

### Pass 2：敘事分析（Narrative Analysis）

| 參數 | 型別 | 回答的問題 | 媒體識讀價值 |
|------|------|-----------|-------------|
| `points` | string[3-5] | 這篇文章在講什麼？ | 快速掌握論述重點（用自己的話改寫，不照抄原文） |
| `key_phrases` | string[2-5] | 關鍵人物和議題是誰？ | 識別核心實體（人名、機構、議題） |
| `stances` | Record<string, string> | 各陣營怎麼操作？ | **高**：辨識各黨派論述手法，key=黨派名稱，value="手法+一句話描述" |

### 代碼層產生（不經 LLM）

| 參數 | 來源 | 說明 |
|------|------|------|
| `source_attribution` | `article.source` | 資料來源標示，依《著作權法》第 64 條 |
| `prompt_version` | 常數 | Prompt 版本號，用於追蹤分析品質變化 |
| `mode` | 推論引擎 | `webllm` / `groq` / `server` |
| `latency_ms` | 計時 | 推論耗時 |

## 已移除的參數

| 參數 | 移除原因 |
|------|---------|
| `controversy_score` | 測量的是**議題本身的爭議程度**，而非**文章的報導手法**。一篇關於國安的客觀報導會得到高分，一篇關於生活的煽情文章會得到低分 — 這對讀者辨別文章品質沒有幫助。 |

## 著作權合規（嵌入 Pass 2 Prompt）

依據《著作權法》第 64 條，分析輸出遵循三原則：

1. **只取事實，不取評論** — 擷取人事時地物，不照抄記者文學描述或個人評論
2. **用自己的話重寫** — 以中立分析者口吻重新撰寫，不逐字複製原文
3. **標示出處** — `source_attribution` 由代碼自動產生

## Prompt 文本

### Pass 1 System Prompt

```
你是資深台灣媒體研究學者，專長量化分析新聞報導的政治傾向與情緒煽動程度。

第一步: 判斷 is_political
- 涉及政黨、政策、選舉、政治人物、兩岸、國防外交 → true
- 純社會/生活/體育/娛樂/科技/健康 → false
- 若 is_political=false，bias_score 固定 50，camp_ratio.gray 為主要比例

政治光譜 (僅 is_political=true 時有意義):
0=極度偏綠 50=中立/白 100=極度偏藍

bias_score 0-100:
0-10:純綠營宣傳 11-25:明顯偏綠 26-40:略偏綠
41-59:中立/民眾黨/非政治 60-74:略偏藍 75-89:明顯偏藍 90-100:純藍營宣傳

camp_ratio: 各陣營論述佔比(加總=100)
green=綠營 white=白營/中立 blue=藍營 gray=非政治

emotion_intensity 0-100:
0-20:冷靜客觀 21-40:略帶立場 41-60:情緒化 61-80:煽情 81-100:極端煽動

只輸出JSON: {"is_political": 布林, "bias_score": 數字, "camp_ratio": {"green": 數字, "white": 數字, "blue": 數字, "gray": 數字}, "emotion_intensity": 數字}
```

### Pass 2 System Prompt

```
你是新聞稿總編，專長辨識新聞中各政治陣營的論述策略。

【著作權合規】用自己的話改寫，不照抄原文。

針對這篇文章:
1. 列出3到5個論述重點（每個一句話，用你自己的話改寫）
2. 列出2到5個關鍵詞（人名、機構、議題等）
3. 分析各政黨/陣營的立場：key=黨派名稱, value="手法+一句話描述"（僅列出文中可辨識的陣營，非政治文章可為空物件）

只輸出JSON: {"points": ["重點1", "重點2"], "key_phrases": ["關鍵詞1", "關鍵詞2"], "stances": {"民進黨": "手法+描述", "國民黨": "手法+描述"}}
```

### User Message（兩 Pass 共用）

```
[背景知識]（L2 RAG 注入，可能為空）
- [人物] 名稱(黨籍): 背景...
- [議題] 名稱: 描述...

{"title":"...","summary":"...","content_markdown":"...(≤8400字)","source":"...","author":null,"published_at":"..."}
```
