# PowerReader — AI代理識讀新聞立場分析平台

**跨越同溫層，看見不同立場的報導。**

PowerReader 是一個去中心化的新聞立場分析工具。透過 AI 技術，幫助你了解台灣不同媒體的報導角度與立場偏向 — 所有分析都在你自己的裝置上完成，新聞內容不會上傳到任何伺服器。

## 為什麼需要 PowerReader？

台灣擁有高度自由的媒體環境，但也意味著同一則新聞，不同媒體可能呈現截然不同的面貌。

我們每天接收的資訊被演算法篩選，不知不覺形成「同溫層」。你看到的觀點，可能只是整個光譜的一小段。

PowerReader 想做的事很簡單：**讓你看見完整的光譜。**

- 這篇報導偏向哪個陣營？偏了多少？
- 同一件事，不同媒體怎麼說？
- 哪些事件只有單一立場的媒體在報導？

---

## 你可以用它做什麼

### 瀏覽今日新聞熱點

打開首頁就能看到今天的重要事件，每則新聞都標示了立場分析結果。你可以依類別篩選（政治、社會、國際、兩岸、財經、科技、娛樂⋯），快速掌握不同議題的媒體報導傾向。

### 比較不同媒體的報導角度

同一事件被 20 多家台灣媒體報導時，每家的立場可能天差地遠。PowerReader 將它們並排呈現，用視覺化的方式讓你一眼看出哪家偏綠、哪家偏藍、哪家相對中立。

涵蓋的媒體包括：中央社、自由時報、聯合報、公視新聞、風傳媒、ETtoday、TVBS、三立新聞等。

### 發現報導盲區

有些事件只被特定立場的媒體報導，另一邊卻完全沉默。PowerReader 自動偵測這些「資訊盲區」，提醒你注意可能被遺漏的觀點：

- 只有泛綠媒體報導的事件
- 只有泛藍媒體報導的事件
- 缺乏中立觀點的議題
- 報導比例嚴重失衡的話題

### 查閱政治脈絡知識庫

808 筆以上由社群共建的政治人物、社會議題、歷史事件資料。你可以查看各黨（民進黨、國民黨、民眾黨、時代力量）對同一議題的立場比較，幫助你在閱讀新聞時擁有更完整的背景脈絡。

## 立場光譜怎麼看

PowerReader 用一條 0 到 100 的光譜來呈現每篇報導的政治立場：

```
偏綠 ◄──────────── 中立 ────────────► 偏藍
0        25        50        75       100
```

除了單一分數，每篇報導還會分析「誰的聲音在主導」：

| 陣營 | 代表 |
|------|------|
| 🟢 綠 | 民進黨及泛綠陣營的論述 |
| 🔵 藍 | 國民黨及泛藍陣營的論述 |
| ⚪ 白 | 民眾黨、中立或跨黨派觀點 |
| ⬜ 灰 | 非政治性內容 |

還有「情緒煽動度」指標（0–100），讓你判斷報導是冷靜客觀還是刻意煽情。

## AI 分析如何運作

PowerReader 使用 [WebLLM](https://github.com/mlc-ai/web-llm) 技術，直接在你的瀏覽器中運行 AI 模型（[Qwen3](https://github.com/QwenLM/Qwen3)-8B），**不需要任何安裝**。

每篇報導經過兩輪分析：

| 輪次 | 做什麼 | 約需時間 |
|------|--------|---------|
| **第一輪** | 量化評分 — 立場分數、陣營比例、情緒煽動度 | ~8 秒 |
| **第二輪** | 敘事解析 — 關鍵論述摘要、涉及的政黨立場 | ~5 秒 |

**分析透明化**：你可以查看 AI 分析時使用的完整背景知識與評分框架，理解分析結果從何而來。每筆分析都附有推論指紋，確保結果的完整性與可追溯性。

> 首次使用需下載一次 AI 模型（約 4.3GB）。下載後，所有分析都在你的裝置上進行，速度快、無需網路、完全私密。不支援本地推論的裝置會自動回退至伺服器端處理。

PowerReader 不靠廣告、不靠企業贊助、不由任何政治立場主導。它由使用者的算力驅動：

```
你分析一篇報導 → 獲得點數獎勵 → 分析結果匯入全網
  → 更多人看見不同觀點 → 更多人加入 → 飛輪越轉越快
```

### 點數與獎勵

- 每次分析可獲得 **0.1–0.5 點數**

- 每日免費配額，用完後仍可繼續分析（不獲得點數）

## 技術架構

<details>
<summary>點擊展開技術細節（給開發者）</summary>

```
┌─────────────┐     WebGPU      ┌──────────────┐
│  瀏覽器 PWA  │ ◄────推論────► │ Qwen3-8B 模型 │
│  SvelteKit 5 │                │  (4.3GB 本地)  │
└──────┬───────┘                └──────────────┘
       │ 僅結構化結果
       ▼
┌──────────────┐    ┌─────┐    ┌─────┐
│  Cloudflare   │───►│ D1  │    │ R2  │
│  Workers API  │    │(SQL)│    │(CDN)│
└──────────────┘    └─────┘    └─────┘
```

| 層級 | 技術 |
|------|------|
| 前端框架 | SvelteKit 2 + Svelte 5 + TypeScript |
| AI 推論 | WebGPU + [WebLLM](https://github.com/mlc-ai/web-llm) (MLC) |
| 語言模型 | [Qwen3](https://github.com/QwenLM/Qwen3)-8B (4-bit 量化, 4.3GB) |
| 部署環境 | Cloudflare Pages (靜態) + Workers (API) |
| 資料庫 | Cloudflare D1 (SQLite) |
| 物件儲存 | Cloudflare R2 |
| 測試 | Vitest (1,342 tests) + Playwright E2E |

### 雙輪推論架構

- **Pass 1（量化）**：提取立場分數、陣營比例、情緒煽動度
- **Pass 2（敘事）**：生成關鍵論述摘要與政黨立場標註
- 文章內容截斷至 8,400 字（約 13K tokens），確保落在模型上下文窗口安全範圍
- 知識庫條目動態注入作為分析背景（RAG）

### 品質驗證管道

每筆分析提交前經過四道驗證：
1. **格式檢查**：JSON 結構完整性
2. **範圍驗證**：所有分數落在有效區間
3. **一致性檢查**：與使用者歷史分析比對
4. **重複防護**：同篇報導不重複分析

</details>
