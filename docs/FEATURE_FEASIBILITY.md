## 文件目的

此文件是 **PowerReader v2.0 所有功能的可行性分類報告**。
將 27 項功能分為三級：A 級 (資料已存在)、B 級 (需新增元件)、C 級 (需新 AI 能力)。
C 級功能全部有降級替代方案。

**核心哲學**: 分析結果是文章的一部分，不是工具。

## 功能可行性分類

### 🟢 A 級 — 資料已存在，只需前端顯示 (12 項)

> 這些功能的資料已在 D1/R2/Vectorize 中，只需寫前端 UI。

| # | 功能 | 資料來源 | 說明 |
|---|------|---------|------|
| A1 | **首頁文章 + 光譜色條** | `articles.bias_score` + `bias_bar.js` 元件 | 已有元件，改為事件卡片上顯示 |
| A2 | **立場雷達圖** | 從 `bias_score` 推導 `getCampFromScore()` | enums.js 已有函式，前端用 Canvas/SVG 畫 |
| A3 | **分析痕跡 — 分析分數** | `analyses.bias_score`, `analyses.controversy_score` | D1 已存，API 已回傳 |
| A4 | **分析痕跡 — 品質數據** | `analyses.quality_gate_result`, `analyses.quality_scores` | D1 已存，JSON 格式 |
| A5 | **分析痕跡 — Prompt 版本** | `analyses.prompt_version` | D1 已存 (如 "v2.0.0") |
| A6 | **分析痕跡 — 最後更新** | `articles.updated_at` | D1 已存 |
| A7 | **分析痕跡 — 總新聞來源數** | `event_clusters.source_count` 或 `COUNT(DISTINCT source)` | 從聚類表查詢 |
| A8 | **原始發布時間** | `articles.published_at` | D1 已存，前端已顯示 |
| A9 | **關鍵字** | `analyses.key_phrases` (JSON array) | D1 已存，Qwen 輸出的 1-10 個關鍵詞 |
| A10 | **新聞發布時間軸** | 同一事件聚類中各文章的 `published_at` | 從 `event_cluster_members` JOIN `articles` 排序 |
| A11 | **類似新聞主題** | `event_clusters` + Jaccard 聚類 | 已設計，zero neuron 成本 |
| A12 | **新聞盲點區塊** | `blindspot_events` 表 (已設計) | 規則式偵測，無 AI 成本 |

**小計**: 12 項，全部用現有資料，**純前端工作**。

---

### 🟡 B 級 — 需要新增元件/表/邏輯，但不需要新 AI (11 項)

> 這些功能需要新的前端元件、新 D1 表、或新 API 端點，但不依賴新的 AI 模型。

| # | 功能 | 需要什麼 | 工作量 |
|---|------|---------|--------|
| B1 | **文章底部進度條 + 完成通知** | Web Worker 推理佇列 + 進度回報 + toast 通知 (+1/+1) | 中 |
| B2 | **硬體偵測 + benchmark** | WebGPU detect + WebLLM 短文推理計時 + localStorage 存結果 (零安裝) | 小 |
| B3 | **背景自動分析 (Mode A)** | 文章打開時自動觸發 WebLLM Service Worker 雙 Pass 推理，不需使用者按按鈕。改 `analyze-engine.js` 為自動模式 | 中 |
| B4 | **首頁快取文章圖片** | Crawler 提取 `og:image` → D1 新增 `articles.thumbnail_url` 欄位 → 前端 `<img>` | 中 |
| B5 | **分析痕跡 — 誰貢獻** | `analyses.user_hash` 已存在 (匿名化)，前端顯示「N 位公民分析師」 | 小 |
| B6 | **分析痕跡 — 使用什麼模型** | 新增 `analyses.model_name` 欄位 (目前只有 Qwen3-4B 但未來可能多模型) | 小 |
| B7 | **分析痕跡 — Prompt 內容可查看** | L1 是靜態文字 (可直接顯示)，L2 來自 `knowledge_ids` → Vectorize 查回 | 中 |
| B8 | **分析分享 (社群平台)** | Web Share API + 社群分享連結 (FB/LINE/Twitter) | 小 |
| B9 | **回饋機制** | 新 D1 表 `article_feedback` (user_hash, article_id, type, created_at) + API + 前端 👍👎 | 中 |
| B10 | **個人化偏好 (多看/少看)** | `user_reading_history` (已設計) + 新 `user_preferences` 表 + 推薦邏輯 | 大 |
| B11 | **+1 代幣動畫** | 獎勵系統已有 (`reward_dedup`, `users.total_points_cents`)，缺前端通知 UI | 小 |

**小計**: 11 項，需要 **後端 + 前端** 配合。

---

### 🔴 C 級 — 需要新 AI 能力或重大基礎建設 (4 項)

> 這些功能超出目前 Qwen 4B 的能力，或需要尚未建立的資料源。

| # | 功能 | 為什麼做不到 | 未來怎麼解 |
|---|------|-------------|-----------|
| C1 | **三種立場重點摘要** | 完整版需按綠/白/藍各生成摘要，需 server-side LLM (7B+) | 先做降級版 + 備註未來升級 |
| C2 | **AI 洞察分析** | "為什麼這篇偏綠" 需要高品質 reasoning。目前 Qwen reasoning 10-200 字，品質不穩定 | 短期: 顯示 reasoning 原文 + 標註參考 |
| C3 | **相關條文/法規** | 需要完整的台灣法規資料庫 + NER + 法規匹配邏輯 | 遠期: 全國法規資料庫 API + NER |
| C4 | **事實性驗證** | 需要即時事實資料庫 + 比 4B 聰明很多的模型 | 非常遠期: 70B+ 模型 + fact-check 整合 |

**小計**: 4 項，**目前不實作**，但有降級替代方案。

## C 級功能的降級替代方案

| 原始功能 | 降級版本 | 需要什麼 | 降級後等級 |
|---------|---------|---------|-----------|
| C1 三方摘要 | **三方標題聚合**: 從各營陣文章取現有 `summary` 顯示 | 只需 SQL query (camp + event_cluster_members) | 🟢 A 級 |
| C2 AI 洞察 | **顯示 Qwen reasoning 原文** + 標註「AI 分析參考」 | `analyses.reasoning` 已存在 | 🟢 A 級 |
| C3 相關條文 | **知識庫條目顯示**: 展示 Vectorize 匹配到的知識條目 | `knowledge_ids` 已存在 | 🟢 A 級 |
| C4 事實性 | **不顯示**，或顯示「事實性驗證：尚未支援」灰色 placeholder | 無 | 不做 |

**用降級方案後**: C1/C2/C3 都變成 A 級（資料已存在），C4 不做。

## 實作優先順序

### Phase 1 — 核心閱讀體驗 (A 級為主)

讓使用者打開就能看到有價值的內容。

| 項目 | 類型 | 工作 |
|------|------|------|
| 事件卡片 + 光譜色條 | A1 | 首頁改為事件聚合 |
| 文章內頁立場分析區塊 | A2, A3 | 雷達圖 + 分數 |
| 三方標題聚合 (降級版) | C1→A | 從各營陣文章取 summary |
| 關鍵字顯示 | A9 | 從 key_phrases 渲染 tag |
| 時間軸 | A10 | 水平時間線元件 |
| 新聞盲點區塊 | A12 | 首頁獨立 section |
| AI 分析參考 (降級版) | C2→A | 顯示 reasoning 原文 |
| 知識庫參考 (降級版) | C3→A | 顯示匹配知識條目 |
| 分析透明度面板 | A4-A7, B5 | 可展開的 metadata 區塊 |

### Phase 2 — 自動化推理 + 互動 (B 級)

讓桌面使用者的推理無縫運作。

| 項目 | 類型 | 工作 |
|------|------|------|
| 硬體偵測 + benchmark | B2 | WebGPU detect + WebLLM 計時 |
| 背景自動分析 | B3 | 重構 analyze 為自動模式 |
| 進度條 + 完成通知 | B1, B11 | 底部進度條 + toast |
| 文章圖片 (og:image) | B4 | Crawler 擴充 + D1 + 前端 |
| 分享按鈕 | B8 | Web Share API |
| 回饋 👍👎 | B9 | 新表 + API + UI |

### Phase 3 — 個人化 + 生態

| 項目 | 類型 | 工作 |
|------|------|------|
| 模型名稱記錄 | B6 | D1 ALTER + 前端 |
| Prompt 內容展示 | B7 | 擴充知識面板 |
| 個人化偏好 | B10 | 完整推薦系統 |

### 不做 / 遠期

| 項目 | 原因 |
|------|------|
| 事實性驗證 (C4) | 需要即時事實 DB + 70B+ 模型 |
| 完整三方摘要生成 (C1 完整版) | 需要 server-side LLM |
| 相關法規條文 (C3 完整版) | 需要法規 DB + NER |
| AI 深度洞察 (C2 完整版) | 需要更強模型 |
