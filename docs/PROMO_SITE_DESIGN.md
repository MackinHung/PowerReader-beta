# PowerReader 宣傳網站設計文件

> **目的**: 為 PowerReader 設計一個獨立的宣傳/介紹型網站，讓訪客快速理解專案定位、技術原理、開發者資訊與未來方向。
> **目標受眾**: 台灣一般新聞讀者 → 公民科技社群 → 技術開發者 → 媒體研究者
> **設計原則**: 三大核心精神 — **去中心化 / 媒體識別 / 公開透明**
> **維護者**: 專案負責人
> **版本**: v2.0
> **建立日期**: 2026-03-09

---

## 受眾分層規則 (CRITICAL)

本網站分為兩層受眾，**嚴禁混用**：

| 層級 | 頁面 | 讀者 | 語調 |
|------|------|------|------|
| **Layer A — 面向讀者** | 首頁、開發者、未來計畫 | 一般新聞讀者、非技術人員 | 白話、零術語、比喻優先 |
| **Layer B — 技術好奇** | 文件 | 公民科技社群、開發者、研究者 | 可用專有名詞、可附規格表 |

### Layer A 禁用詞表 (首頁 / 開發者 / 未來計畫)

以下詞彙**絕對不可**出現在 Layer A 頁面：

| 禁用 | 替代說法 |
|------|---------|
| Qwen3-8B | 「AI 模型」或「分析引擎」 |
| WebLLM / WebGPU | 「瀏覽器內建的加速技術」 |
| VRAM / GPU | 「你電腦的運算能力」或「你的裝置」 |
| SHA-256 / 雜湊 | 「防竄改驗證」或「數位指紋」 |
| AGPL-3.0 | 「完全開源授權」 |
| RAG / Retrieval-Augmented Generation | 「知識注入」或「背景資料補充」 |
| ALBERT / bge-m3 / bge-small-zh | 「分類模型」或「篩選系統」 |
| F1 分數 / cosine / embedding | 不提，或用「準確率」 |
| Cloudflare / Workers / D1 / KV / R2 | 「雲端伺服器」 |
| PWA / Service Worker | 「可安裝的網頁應用」或直接不提 |
| Prompt / L1 / L2 / L3 | 「AI 的分析指令」或「分析依據」 |
| WebGPU benchmark | 「硬體偵測」 |
| Pass 1 / Pass 2 | 「兩輪分析」 |
| OAuth | 「Google 帳號登入」 |
| Vanilla JS / IndexedDB / OPFS | 不提 |
| tokens / context window | 不提 |

### Layer A 允許的最大技術深度

- 可以說「AI」「模型」「開源」
- 可以說「瀏覽器裡分析」「不上傳資料」「本地端」「雲端」
- 可以說「18+ 新聞來源」「每 2 小時」「約 7 秒」
- 比喻可以用 (「中央空廚」「考試帶小抄」)

**引言** (流程圖上方):

> 我們幫助有品質的報導被更多人看見，也更細緻地辨識哪些文章值得你花時間閱讀。

**文案**:

> **Section 標題**: 它怎麼運作？

> **步驟 1**: 自動收集新聞
> 我們每 2 小時從 18 個以上台灣新聞來源自動收集最新文章，涵蓋不同政治立場的媒體，經過篩選和整理。

> **步驟 2**: 補充背景知識
> 系統根據文章主題，自動搜尋相關背景資訊——政治人物、議題脈絡、媒體特徵——讓 AI 做出更準確的判斷。

> **步驟 3**: 你的電腦負責分析
> AI 模型直接在你電腦上運行——新聞內容不會離開你的裝置。每篇約 7 秒完成分析。

> **步驟 4**: 透明化的結果
> 分析結果包含立場分數、爭議度、推理過程、以及 AI 參考的所有知識來源。你可以逐項檢視，決定是否信任。

**設計細節**:
- 四個步驟用虛線箭頭連接
- 步驟 1、2 標記「雲端」(灰色 badge)，步驟 3、4 標記「你的電腦」(青綠色 badge)
- 這個「雲端 → 你的電腦」的分界線是重要視覺元素——用一條漸層虛線分隔
- 步驟序號使用圓形背景 + 數字

### Section 5: 台灣為什麼需要這個？ (Problem Statement)

**視覺**: 白色背景，左文右圖 (或左圖右文交替)

**文案**:

> **Section 標題**: 台灣的新聞環境

> **段落 1 — 問題**:
> 台灣擁有蓬勃的新聞自由，但同一則事件在不同媒體筆下，可能呈現截然不同的故事。對讀者來說，辨別報導角度需要大量的媒體素養與時間——而大多數人只會看到演算法推薦的單一觀點。

> **段落 2 — 現有工具的不足**:
> 國際上有 Ground News、AllSides 等媒體偏見分析工具，但它們都不支援繁體中文，也不理解台灣獨特的政治光譜。台灣目前沒有同等定位的工具。

> **段落 3 — 我們的回答**:
> PowerReader 是第一個專為台灣新聞環境設計的立場分析工具。它不預設立場、不依賴人工標注、不追蹤你的閱讀行為。它只做一件事：幫助你看見同一則新聞的不同面向。

**設計細節**:
- 右側搭配一個簡化的插圖：同一則新聞標題，三個不同顏色的框框顯示不同角度
- 或者：一個人物剪影看著三個不同顏色的螢幕

### Section 3: 資料收集與處理

**視覺**: 帶編號的展開式手風琴 (Accordion) 或垂直時間軸

**3.1 — 新聞收集**

> **標題**: ① 新聞收集 — 每 2 小時，自動巡邏
>
> PowerReader 的爬蟲系統每 2 小時自動從 18 個以上台灣新聞來源收集最新文章。來源涵蓋泛綠、泛藍、中立、公共媒體等不同立場光譜。
>
> 爬蟲嚴格遵守 robots.txt 規範，每次請求間隔至少 2 秒，並使用公開透明的 Bot 識別碼 (`MediaBiasBot/1.0`)。我們不爬取付費牆後的內容，也不儲存原始 HTML。
>
> 來源清單完整公開在 GitHub 上。

**設計**: 旁邊放一個簡化的新聞來源列表圖 (只列幾個代表性的 logo 或名稱，不需要全部)

**3.2 — 議題篩選**

> **標題**: ② 議題篩選 — AI 篩選相關新聞
>
> 並非所有新聞都適合立場分析。系統使用 ALBERT 分類模型 (在 HuggingFace 上公開) 自動判斷文章是否屬於政治、國際、兩岸、社會等相關議題。娛樂、體育、天氣等類別會被自動過濾。
>
> 這個篩選模型的準確率 (F1 分數) 為 0.824，使用 544 篇真實台灣新聞文章微調訓練而成。

**3.3 — 去重處理**

> **標題**: ③ 去重 — 識別同一事件的不同報導
>
> 台灣新聞有大量「通稿改寫」的現象——同一則中央社電稿可能被十幾家媒體轉載。系統使用兩層去重：
> - **精確去重**: SHA-256 雜湊偵測完全相同的文章
> - **語意去重**: AI 嵌入模型 (bge-small-zh) 比對相似度，分辨「同一篇轉載」vs.「同事件不同角度」
>
> 重點不是刪除重複——而是把報導同一事件的文章分組在一起，這正是跨媒體比較的基礎。

**3.4 — 知識注入 (RAG)**

> **標題**: ④ 知識注入 — 讓小模型也有大智慧
>
> 瀏覽器裡的 AI 模型 (Qwen3-8B) 雖然強大，但它的知識有截止日期，也不了解台灣的最新政治動態。
>
> 為了解決這個問題，系統會根據每篇文章的主題，自動從知識庫中搜尋最相關的背景資訊——包括政治人物檔案、黨派關係、議題脈絡、媒體特徵等。這些知識會和文章一起送進 AI 分析。
>
> 這種技術叫做 RAG (Retrieval-Augmented Generation)——你可以把它想成：考試的時候讓 AI 帶小抄。
>
> **所有注入的知識條目，你都可以在分析結果中展開查看。** 我們不做任何黑箱操作。

### Section 4: 瀏覽器內 AI 推理

**視覺**: 重點 Section，使用強調配色 (淺青綠背景或邊框)

**文案**:

> **Section 標題**: 去中心化 AI — 在你的瀏覽器裡運行

> **什麼是 WebLLM？**
>
> WebLLM 是一種讓大型語言模型直接在瀏覽器中運行的技術。它利用 WebGPU (瀏覽器原生的 GPU 加速 API) 讓 AI 模型在你的電腦上執行推理，完全不需要安裝任何軟體——不需要 Ollama、不需要 CUDA 轉換、不需要任何複雜的環境設定。
>
> PowerReader 使用的模型是 Qwen3-8B (80 億參數)——它在首次使用時會下載約 4.5GB 到瀏覽器快取中，之後直接從本地載入。

> **為什麼要在本地跑？**
>
> | 雲端 AI 分析 | PowerReader (本地端) |
> |-------------|---------------------|
> | 你的文章上傳到伺服器 | 文章只留在你的瀏覽器 |
> | 伺服器知道你讀了什麼 | 沒有任何人知道你讀了什麼 |
> | 需要依賴單一服務商 | 你的電腦就是你的分析站 |
> | 服務商可以修改結果 | 開源模型 + 開源 Prompt = 可驗證 |

> **雙 Pass 分析架構**
>
> 每篇文章會被分析兩次，每次聚焦不同面向：
> - **Pass 1**: 分析立場傾向、偏見指標
> - **Pass 2**: 分析爭議度、關鍵論點
>
> 兩次分析結果合併後，提供更全面、更穩健的判斷。整個過程約 14 秒 (每次 ~7 秒)。

### Section 5: 品質驗證

**文案**:

> **Section 標題**: 四層品質閘門 — AI 也需要被檢驗
>
> 不是每個 AI 分析結果都是正確的。PowerReader 用四層自動品質驗證確保結果可信：
>
> 1. **格式驗證**: 分析結果是否為有效的結構化資料
> 2. **範圍驗證**: 分數是否在合理範圍內 (0-100)
> 3. **一致性驗證**: 分數與分類是否互相吻合
> 4. **去重驗證**: 是否與已有分析結果高度重複
>
> 只有通過全部四層驗證的分析才會被採納。通過率約 60-70%——我們寧可拒絕不確定的結果，也不接受品質不佳的分析。

**視覺**: 深靛藍背景，白色文字 + 網絡動畫

### Section 7: Prompt 與知識庫透明化

**文案**:

> **Section 標題**: 完全透明 — 連 AI 的指令都公開
>
> 許多 AI 產品把 Prompt (提示詞) 視為商業機密。PowerReader 不是。
>
> 我們的 Prompt 採用三層架構：
> - **L1 — 靜態指令 (~300 tokens)**: 定義分析框架、偏見特徵、分數錨點。完全公開在 GitHub 上。
> - **L2 — 動態知識 (~200-800 tokens)**: 根據文章主題自動注入的背景資訊。每筆分析結果都可展開查看。
> - **L3 — 文章輸入**: 新聞原文 + 輸出格式指令。
>
> 總 Prompt 長度嚴格控制在模型上下文窗口的 40% 以內，確保 AI 有足夠空間思考。
>
> **你可以在每筆分析結果中點擊「分析透明度」面板，查看 AI 讀了什麼、被告知了什麼、以及它是怎麼推理出這個結論的。**

## 文案語調指南 (Tone of Voice)

### 核心原則
1. **平易近人**: Layer A 頁面零術語。Layer B 的術語要附帶通俗解釋 (如 RAG 比喻為「考試帶小抄」)
2. **誠實不誇大**: 通過率 60-70% 就說 60-70%，不說「高準確率」
3. **不恐嚇讀者**: 不用「你的資料正在被監控」這種恐慌式行銷
4. **展示而非宣稱**: 說「你可以在透明度面板中查看」比「我們很透明」有說服力
5. **台灣在地感**: 使用台灣讀者熟悉的媒體名稱、政治語境

### 避免的用語
- ❌ 「革命性」「顛覆」「前所未有」→ 過度行銷
- ❌ 「客觀」「中立」「公正」→ 沒有任何工具是完全客觀的
- ❌ 「偏見矯正」→ 我們不矯正任何東西，只揭示不同角度
- ❌ 「假新聞偵測」→ PowerReader 不判斷真假，只分析立場

### 推薦的用語
- ✅ 「揭示不同角度」「看見不同面向」
- ✅ 「幫助你自己判斷」
- ✅ 「透明」「可驗證」「可檢視」
- ✅ 「公民科技」「開源」「社群驅動」
